# Programmable Timer with Interrupt Controller (Verilog)

## Overview

This project implements a **Programmable Timer integrated with an Interrupt Controller** using Verilog. The timer counts clock cycles based on a programmable value and generates an interrupt when the count expires. The interrupt controller manages multiple interrupt sources and sends prioritized interrupt signals to the CPU.

This design demonstrates important digital design concepts including:

* Programmable timers
* Interrupt handling
* Priority encoding
* Sequential logic design
* Hardware event signaling

---

## System Architecture

The system consists of three main functional blocks:

```
Clock
  │
  ▼
Programmable Timer
  │
  ▼
Timer Interrupt Request
  │
  ▼
Interrupt Controller
  │
  ▼
CPU Interrupt Acknowledge
```

The **programmable timer generates interrupts**, and the **interrupt controller prioritizes multiple interrupt sources** before notifying the CPU.

## Inputs

| Signal             | Description                |
| ------------------ | -------------------------- |
| `clk`              | System clock               |
| `rst`              | Reset signal               |
| `user_choice[3:0]` | Timer count value          |
| `keyboard_irq`     | Keyboard interrupt request |
| `sensor_irq`       | Sensor interrupt request   |

---

## Outputs

| Signal      | Description                      |
| ----------- | -------------------------------- |
| `counter`   | Current timer counter value      |
| `timer_irq` | Interrupt generated by timer     |
| `inr_id1`   | Timer interrupt ID               |
| `inr_id2`   | Keyboard interrupt ID            |
| `inr_id3`   | Sensor interrupt ID              |
| `ack[1:0]`  | CPU interrupt acknowledge signal |

---

## Interrupt Priority

The interrupt controller resolves simultaneous interrupts using the following priority:

```
Timer Interrupt   >   Keyboard Interrupt   >   Sensor Interrupt
```

If multiple interrupts occur at the same time, the **highest priority interrupt is serviced first**.

---

## Timer Operation

1. The user loads a programmable timer value using `user_choice`.
2. The internal counter increments every clock cycle.
3. When the counter equals the programmed value:

   * `timer_irq` is asserted.
   * The counter resets and begins counting again.

This creates a **periodic timer interrupt**.

---

## Interrupt Handling

When an interrupt occurs:

* The interrupt controller detects the active interrupt source.
* The corresponding interrupt ID (`inr_id`) is generated.
* The CPU receives an interrupt acknowledge signal (`ack`).

Example mapping:

| Interrupt | Acknowledge |
| --------- | ----------- |
| Timer     | `01`        |
| Keyboard  | `10`        |
| Sensor    | `11`        |
